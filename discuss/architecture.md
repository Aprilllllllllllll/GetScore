# 成绩查询系统 — 架构说明

## 概述

自动监控广西师范大学教务系统的成绩变化，登录→抓取→解析→存储→通知 的全流程自动化。

## 技术栈

| 组件 | 技术选型 |
|------|---------|
| 语言 | Python 3.8+ |
| 包管理 | uv |
| HTTP 请求 | requests |
| RSA 加密 | pycryptodome |
| 数据存储 | SQLite |
| 定时调度 | APScheduler |
| 消息推送 | Server酱 Turbo (https://sct.ftqq.com/) |
| 配置管理 | python-dotenv |

## 数据流

```
  └─────────┘                  └────────────────┘
      │
      ▼
  ┌────────┐   成绩判断/解析   ┌──────────────┐

```

## 模块职责

| 模块 | 文件 | 行数 | 职责 |

## 挂科判断规则

满足以下任一条件即为挂科：
1. 成绩状态为 `不及格`/`缺考`/`作弊`/`取消资格`/`违纪`（优先级最高）
2. 综合成绩（zhcj）为有效数字且 < 60
3. 绩点（jd）为 0 且综合成绩 < 60

## 差异检测

- 唯一键：`(raw_course_code, semester)` — 课程编号 + 学期
- 对比维度：score、grade_point、status、is_fail 等关键字段
- 首次出现 → 标记为 "新增"
- 关键字段变化 → 标记为 "更新"（包含旧分数）

## 安全注意事项

1. `.env` 文件包含明文密码，已加入 `.gitignore`
2. RSA 公钥可能随时间过期，登录失败时需重新抓包获取
3. 教务系统 Cookie 有效期约 2-4 小时，调度器会自动重新登录
4. Server酱 SendKey 属于敏感信息，不要提交到版本控制
